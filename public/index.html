what makes smile:D different

scope exactly what the agent can access

create a context per project and enable only the connectors it needs. set which tools it can call and which knowledge it can read. the agent never exceeds its scope.

build connectors for any tool

smile:D connectors are just code that expose tools. build your own for internal APIs, databases, or proprietary systems. a connector SDK is coming soon.

use the models you want, your way

switch between chat, reasoning, and vision models per task. when a new model comes out, plug it in and try it. no vendor lock-in, no forced interface.

approve every action

the agent pauses before creating, updating, or deleting anything. you see a preview of exactly what it will do, then accept or refuse with one click.
